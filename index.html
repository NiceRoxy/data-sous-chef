😤 The Problem

Research data is messy.
Cleaning it shouldn't be your job.

Researchers spend 60–80% of their time wrangling data instead of analysing it. The tools that exist either require coding expertise or demand you upload sensitive data to the cloud.

🧩

Inconsistent formats across files

Date columns formatted three different ways. Categories spelled inconsistently. Column names that vary between academic years.

🔐

Can't share data with cloud tools

GDPR, IRB restrictions, and institutional governance policies mean you can't upload raw datasets to third-party platforms — even to get help cleaning them.

💻

Writing the script yourself takes hours

Even if you know Python, writing reliable, annotated cleaning code for a 50,000-row admin dataset is tedious, error-prone, and hard to audit later.

🔒 Privacy by Design

Your data never leaves
your machine. Ever.

Built from the ground up for researchers working under GDPR, IRB restrictions, and institutional data governance. DataSousChief is structurally incapable of seeing your data — because we never ask for it.

  • ✓
    No file upload — you describe your data in plain English, we never see it
  • ✓
    Firebase stores only user-declared metadata — zero raw data values
  • ✓
    Generated scripts run entirely on your local machine at runtime
  • ✓
    Compliant with GDPR, IRB, and institutional data governance policies
  • ✓
    Only generated .py scripts are stored in Firebase Storage
🛡️

GDPR Safe

No personal data is processed or stored. You retain full custody of your research files at all times.

🔬

IRB / Ethics Board Friendly

Institutional Governance Ready
